.gitignore

.gitignore文件使得在commit时，指定文件夹不会被commit。比如文件夹vendor。该文件夹里有我们所需要的库。在多人开发时，不同的开发者的开发环境可能不同，所以所需要的库的版本有可能不同。开发者可修改composer.json文件，然后再命令行执行composer install安装库。

.htaccess

huge/.htaccess用来对路由进行重写,RewriteEngine on是开启重写，RewriteRule将所有路由都重定向到public目录下面。其中(.*)匹配到字符串存储到变量$1。

Session.php

1. session_start()的初始化工作：
（1）读取请求头部Cookie字段的PHPSESSID，假使为abc123.
（2）若读取到Cookie字段的PHPSESSID,创建$_SESSION变量，并从相应的目录中（可以在php.ini中设置session.save_path）读取SESS_abc123（默认是这种命名方式）文件，将字符装在$_SESSION变量中；若没有读取到PHPSESSID这个COOKIE，也会创建$_SESSION变量，同时创建一个sess_abc321(名称为随机值)的session文件,同时将abc321作为PHPSESSID的cookie值返回给浏览器端。

2. session_id()
在开启session_start()之后，session_id()返回PHPSESSID

参考资料：
PHP之session_start()详解

isConcurrentSessionExists()判断是否有同时登陆的账号，过程如下：
在数据库中每个user对应着一个session_id。userA先登陆账号，Session_id是123。一会userB在另一台电脑登陆和userA同一个账号，userB对应的session_id为456。userB在登录的过程中，将自己的session_id(456)和数据库中的session_id(123)进行比对，发现不一样，则判断当前有多个设备登录同一个账号。

参考资料：
如何管理Session(防止恶意共享账号)——理论篇

controller目录

LoginController.php

  1. index(),检查是否已经登录，若没有登录，则跳转到登录页面。
  2. login()，检查csrf_token是否合法。然后调用LoginModel检查用户名和密码是否正确，如果正确则跳转到用户页面user/index，否则返回到登录页面login/index。
  3. logout()退出登录
  4. loginWithCookie()，使用Cookie的remember_me进行登录。如果登录失败则删除cookie，并跳转到登录页面。

model目录

LoginModel.php

1. login()
（1）检查用户名和密码是否为空
（2）调用validateAndGetUser($user_name, $user_password)进行相关检查
（3）检查该用户是否被删除
（4）检查账户是否被停用
（5）重置用户失败登录次数，保存登录时间。
（6）如果用户设置remember_me，则调用setRememberMeInDatabaseAndCookie($user_id)

2. validateAndGetUser($user_name, $user_password)
（1）检查用户登录失败的次数是否>=3,以及最后一次登录时间距离当前时间少于30s。（针对于用户名不正确）
（2）若用户登录没有被锁，则调用UserModel::getUserDataByUsername($user_name),通过用户名从user表中获取用户所有信息。
（3）如果用户不存在，用户登录失败次数+1，添加最后一次登录失败的时间戳。
（4）如果用户名正确，但是登录失败次数超过3次以及最后一次登录时间距离当前时间少于30s,则禁止登录。
（5）判断输入的密码的哈希值和数据库中的哈希值是否相等。
（6）检查用户是否激活
（7）重置用户名不存在的次数为0。

3. setRememberMeInDatabaseAndCookie($user_id)
（1）随机生成token，存入数据库中。
（2）setcookie()

4. logout()
(1)self::deleteCookie($user_id);删除cookie
(2)销毁session变量
(3)更新数据库中的session_id为null

5. deleteCookie()
（1）更新数据库中的user_remember_me_token为null
（2）删除remember_me字段

6. loginWithCookie()
（1）将remeber_me字段切割成$user_id,$token,$hash三部分。检测是否正确。
（2）登录成功，更新session信息和保存登录时间

疑问

1. 类方法static与non-static的区别？
在Application中有这一条语句Request::get('url')，然后再去看Request类中看get方法是static。如果把static去掉的话，就会报错Deprecated: Non-static method Request::get() should not be called statically.这是为什么呢？所以我就去了解了一波。如果是static方法的话，那再调用该方法时，不需要将类初始化，可直接用这种形式classname::method()调用类方法。如果是non-static的话，这需要初始化类，比如：$var = new ClassName(); $var->method();
static是在程序运行时，就加载到内存中的。而non-static的话是需要实例化后才加载到内存中。所以static方法不能调用non-static方法。
